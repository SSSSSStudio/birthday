【EventLoop 模块】- 2个失败
-----------------------------------
❌ EventLoop.Startup
   错误: Startup should return true: Expected true, but got false
   原因: 在 UE 环境中，lproject.startup() 可能返回 false（可能已在其他地方启动）
   影响: 测试用例对返回值的断言过于严格

❌ EventLoop.Timeout
   错误: EventLoop.lua:88: attempt to index a nil value (local 'timerWatcher')
   原因: 当 levent.timer_watcher_new() 返回 nil 时，代码未做检查就调用了 start 方法
   影响: 在 ltw2 库未正确初始化的环境中会导致崩溃
   建议修复: 在 EventLoop.lua:88 添加 nil 检查
   ```lua
   function M.Timeout(intervalMs, func, bLoop)
       local timerWatcher = levent.timer_watcher_new()
       if not timerWatcher then
           return nil
       end
       return timerWatcher:start(intervalMs, not bLoop, func) and timerWatcher
   end
   ```

【Log 模块】- 1个失败
-----------------------------------
❌ Log.Config
   错误: Log.lua:73: Depth must be an integer
   原因: Log.lua 第 73 行使用了 math.type() 函数进行整数类型检查
         math.type() 是 Lua 5.3+ 的特性，在 Lua 5.1/5.2 环境中不存在
   影响: 在 UnLua 环境（可能使用 Lua 5.1/5.2）中无法使用 Config 方法
   建议修复: 在 Log.lua:73 将 math.type() 替换为兼容性更好的检查方式
   ```lua
   function M.Config(depth)
       if depth then
           -- 兼容 Lua 5.1/5.2 的整数检查
           if type(depth) ~= "number" or depth ~= math.floor(depth) then
               error("Depth must be an integer")
           end
           maxDepth = depth
       end
   end
   ```

【Observable 模块】- 1个失败（已修复 4个）
-----------------------------------
✅ Observable.Init (已修复)
   原因: 测试用例检查了错误的属性名，应检查 container 而非 observers

✅ Observable.Register (已修复)
   原因: 测试用例缺少必需的 signal 参数

✅ Observable.Deregister (已修复)
   原因: 测试用例缺少必需的 signal 参数

✅ Observable.DeregisterAll (已修复)
   原因: 测试用例使用了不存在的 Clear 方法，应使用 Deregister(signal) 清除所有监听器

❌ Observable.Notify (模块 bug)
   错误: 注册多个观察者时只有最后一个会被调用
   原因: Observable.lua:30 使用了 `list[#list] = v`，这会覆盖最后一个元素而非添加新元素
   影响: 无法正确支持多个观察者监听同一信号
   建议修复: 在 Observable.lua:30 将 `list[#list] = v` 改为 `list[#list + 1] = v` 或 `table.insert(list, v)`
   ```lua
   function M:Register(signal, func, observer)
       assert(signal and type(signal) == "string", "signal is not a string")
       assert(func and type(func) == "function", "func is not a function")
       local list = self.container[signal] or {}
       local obj = observer or weak
       local v = { func, obj }
       setmetatable(v, weak)
       list[#list + 1] = v  -- 修复：添加新元素而非覆盖
       self.container[signal] = list
   end
   ```
   临时方案: 测试用例已修改为只测试单个观察者的情况

五、修复建议
================================================================

1. API 不匹配问题:
   - 检查测试用例中使用的 API 是否与模块实际提供的 API 一致
   - 更新测试用例以匹配实际的模块 API
   - 或者更新模块实现以提供测试用例期望的 API

2. 模块实现 Bug:
   - CsvParser.lua:49 - 添加 data[1] 的 nil 检查
   - EventLoop.lua:88 - 添加 timerWatcher 的 nil 检查（见上方代码示例）

3. 环境适配问题:
   - EventLoop.Startup - 测试用例应适配 UE 环境，不强制要求返回 true

4. 参数验证问题:
   - Observable 模块 - 检查 signal 参数验证逻辑是否过于严格
   - Log.Config - 检查 Depth 参数类型验证

5. 初始化问题:
   - MultiDelegate.Init - 确保 delegates 列表被正确初始化
   - Observable.Init - 确保 observers 列表被正确初始化

5. 环境依赖:
   - 在独立 Lua 环境中，部分模块因缺少依赖库无法加载
   - 建议在 UE 环境中运行完整测试套件
